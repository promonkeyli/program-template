# mobile-taro

这是一个基于 Taro 框架的现代化移动端应用模板，旨在提供一个开箱即用的多端开发解决方案。

## ✨ 特性

- **跨平台**: 一套代码，多端运行，支持微信小程序、H5、支付宝小程序等。
- **技术栈**: 使用 React、TypeScript、Less 和 Zustand，享受现代前端开发的乐趣。
- **状态管理**: 内置 Zustand 进行全局状态管理，轻量且强大。
- **架构思想**: 采用基于自定义 Hooks 的架构，将业务逻辑与 UI 彻底分离，提高代码的可读性和可维护性。
- **样式规范**: 强制使用 `src/app.less` 中的全局 Less 变量来定义颜色和字体，确保项目视觉风格的统一和可维护性。
- **代码规范**: 集成 ESLint, Stylelint, EditorConfig，确保代码风格一致和高质量。
- **提交规范**: 使用 Husky 和 Commitlint 强制执行规范化的提交消息。
- **环境配置**: 区分开发、生产和测试环境，方便管理不同环境的配置。
- **目录结构**: 清晰、可扩展的模块化目录结构，所有功能单元（页面、组件、服务等）均以文件夹形式组织。

## 📦 安装与启动

本项目使用 `pnpm` 作为包管理器。

1.  **安装依赖**

    ```bash
    pnpm install
    ```
    > **注意**: 如果安装过程中出现 `EPERM` 权限错误，这可能是您本地环境的问题。请尝试检查文件权限或以管理员身份运行终端。

2.  **启动开发环境**

    - 启动微信小程序开发环境：
      ```bash
      pnpm dev:weapp
      ```
    - 启动 H5 开发环境：
      ```bash
      pnpm dev:h5
      ```
    - 其他平台请参照 `package.json` 中的 `scripts` 命令。

3.  **打包构建**

    - 构建微信小程序：
      ```bash
      pnpm build:weapp
      ```
    - 构建 H5 应用：
      ```bash
      pnpm build:h5
      ```

## 📁 目录结构

项目采用模块化的文件夹结构，以实现高度的组织性和可维护性。

```
.
├── config                   # Taro 编译配置
├── src                      # 源码目录
│   ├── app.config.ts        # 全局配置
│   ├── app.less             # 全局样式 (定义通用色系、字体等变量)
│   ├── app.ts               # 入口文件
│   ├── components           # 全局通用组件 (每个组件一个文件夹)
│   ├── pages                # 页面模块 (每个页面一个文件夹)
│   ├── services             # API 服务 (每个业务模块一个文件夹)
│   ├── stores               # 全局状态管理 (每个 store 一个文件夹)
│   └── utils                # 通用工具函数
│       └── http.ts          # HTTP 请求封装
├── package.json             # 依赖管理
└── tsconfig.json            # TypeScript 配置
```
